SDE Intern Fullstack Take-Home: Swag Commerce & Analytics System
📋 Overview
A merchandise vendor needs a fullstack platform where companies can purchase swag in bulk. The system must support a "Buy-Sell" flow, a dynamic coupon system for promotions, and an AI-powered admin assistant to help make data-driven decisions.
Timeline: 48 Hours
Level: SDE Intern (Fullstack)
🚀 The Challenge
1. System Architecture & Data Management
To support a robust commerce experience, you are required to design and implement a scalable data layer and a reliable server-side communication interface.
Information Modeling:
Define a structure for Products that tracks descriptions, stock availability, and tiered pricing.
Develop a Promotional Logic schema to handle unique codes, varying discount types (fixed vs. percentage), and usage constraints like expiration and limits.
Establish an Order History system that logs customer details, transaction totals, applied discounts, and precise timestamps.
Service Interfaces:
Create a retrieval system for the product catalog with support for categorized browsing.
Build a validation engine that checks promotional codes against active cart items to return real-time price adjustments.
Implement a transactional purchase flow that verifies inventory, applies business logic to ensure correct final pricing (including $0 transactions), and maintains data integrity across stock and usage counters.
Develop an analytics aggregator to supply the intelligent assistant with high-level performance metrics, stock alerts, and revenue trends.
2. Coupon System (Business Logic)
Implement logic where specific coupons can make an item completely free ($0).
Ensure a coupon cannot be used if it has expired or reached its usage limit.
The UI must clearly show the "Original Price" vs "Discounted Price" before checkout.
3. Admin AI Copilot (The Intelligent Layer)
Feature: Integrate a chat interface in the Admin Dashboard.
Capability: Allow the admin to ask questions like:
"Which item is running lowest on stock?"
"What was my total revenue today?"
"Is the coupon 'SUMMER50' still valid?"
Tech Hint: Use the Gemini API (Free Tier via Google AI Studio). You should pass a summary of your database stats as "context" to the AI prompt so it can answer correctly.
4. User Portal (The Storefront)
Marketplace: A "buy" experience where users add items to a cart.
Checkout: A field to apply coupons and a "Place Order" button.
🛠 Technical Stack
Frontend: React.js / Next.js + Tailwind CSS.
Backend: Node.js (Express) or Python (FastAPI).
AI: Google Gemini SDK (@google/generative-ai).
Database: PostgreSQL, MongoDB, or SQLite.
🎨 UI/UX Expectations
AI Experience: The chatbot should have a "thinking" state and handle errors gracefully if the API limit is hit.
Pricing Clarity: Visual strike-through on original prices when a coupon is applied.
Responsiveness: Must be mobile-friendly.
